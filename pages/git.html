Git Cheat Sheet

Setup

git config --global user.name "Name" Set your name globally
git config --global user.email "email" Set your email globally
git config --global core.editor vim Set default editor
git config --list Show all config values

Init & Clone

git init Initialise a new repo in current directory
git clone <url> Clone a remote repo
git clone <url> <dir> Clone into a specific folder
git clone --depth 1 <url> Shallow clone — latest commit only

Staging

git status Show working tree status
git add <file> Stage a specific file
git add . Stage all changes
git add -p Interactively stage chunks — very useful
git restore --staged <file> Unstage a file without discarding changes
git restore <file> Discard working directory changes destructive

Commits

git commit -m "message" Commit with a message
git commit -am "message" Stage tracked files and commit in one step
git commit --amend Edit the last commit message or add staged changes rewrites history
git commit --amend --no-edit Add staged changes to last commit, keep message

Branches

git branch List local branches
git branch <name> Create a new branch
git switch <name> Switch to a branch
git switch -c <name> Create and switch in one step
git branch -d <name> Delete a merged branch
git branch -D <name> Force delete a branch destructive

Merge & Rebase

git merge <branch> Merge branch into current
git merge --no-ff <branch> Merge and always create a merge commit
git rebase <branch> Rebase current branch onto another rewrites history
git rebase -i HEAD~n Interactive rebase — squash, reorder, edit last n commits
git rebase --abort Cancel an in-progress rebase

Remote

git remote -v List remotes with URLs
git remote add origin <url> Add a remote
git fetch Download remote changes without merging
git pull Fetch and merge remote changes
git pull --rebase Fetch and rebase instead of merge cleaner history
git push -u origin <branch> Push and set upstream tracking
git push --force-with-lease Safer force push — fails if remote has new commits

Log & Diff

git log --oneline --graph --all Compact visual branch history
git log -p <file> Show commit history with diffs for a file
git diff Show unstaged changes
git diff --staged Show staged changes
git diff <branch1>..<branch2> Diff between two branches
git blame <file> Show who changed each line

Stash

git stash Stash current changes
git stash push -m "message" Stash with a label
git stash list List all stashes
git stash pop Apply latest stash and remove it
git stash apply stash@{n} Apply a specific stash without removing it
git stash drop stash@{n} Delete a specific stash

Undoing

git revert <hash> Create a new commit that undoes a commit — safe for shared branches
git reset --soft HEAD~1 Undo last commit, keep changes staged
git reset --mixed HEAD~1 Undo last commit, keep changes unstaged
git reset --hard HEAD~1 Undo last commit, discard all changes destructive
git reflog History of HEAD — recover lost commits lifesaver

Cherry-pick & Tags

git cherry-pick <hash> Apply a specific commit to current branch
git cherry-pick <hash1>..<hash2> Apply a range of commits
git tag <name> Create a lightweight tag
git tag -a <name> -m "msg" Create an annotated tag
git push origin --tags Push all tags to remote

Search

git grep "pattern" Search working tree for a pattern
git log -S "string" Find commits that added or removed a string
git log --all --grep="message" Search commit messages
git bisect start Start binary search to find a bad commit

Undo Last Commit (Keep Changes)

Easy
Scenario: You committed too early and want to add more changes to the commit.
Step 1: Undo the commit, keep changes staged
git reset --soft HEAD~1
Step 2: Make your additional changes
git add <files>
Step 3: Commit again
git commit -m "Complete feature"
Alternative: Use git commit --amend to add staged changes to the last commit without creating a new one.

Made Commits on Wrong Branch

Medium
Scenario: You made commits on main but they should be on a feature branch.
Step 1: Create a new branch (keeps commits)
git branch feature-branch
Step 2: Reset main to before your commits
git reset --hard origin/main
Step 3: Switch to your feature branch
git switch feature-branch
Warning: Only use --hard if you're sure. It discards uncommitted changes.

Resolve Merge Conflicts

Medium
Scenario: You tried to merge and got conflicts.
Step 1: See which files have conflicts
git status
Step 2: Open conflicted files and look for markers 
<<<<<<< HEAD
Your changes
=======
Their changes
>>>>>>> branch-name
Step 3: Edit files to resolve conflicts
Remove conflict markers and keep the code you want
Step 4: Mark as resolved
git add <resolved-files>
Step 5: Complete the merge
git commit
Abort: If you want to cancel the merge: git merge --abort

Recover Deleted File

Easy
Scenario: You deleted a file and want it back.
If not yet committed:
git restore <file>
If already committed (get from last commit):
git checkout HEAD~1 -- <file>
Find when file was deleted:
git log --all --full-history -- <file>

Undo a Commit That's Been Pushed

Medium
Scenario: You pushed a bad commit and need to undo it safely.
Step 1: Find the commit hash
git log --oneline
Step 2: Create a new commit that undoes it
git revert <commit-hash>
Step 3: Push the revert
git push
Why revert? Using git revert is safer than git reset for shared branches because it preserves history.

Save Work to Switch Branches

Easy
Scenario: You're working on something but need to switch branches urgently.
Step 1: Stash your changes
git stash push -m "WIP: feature description"
Step 2: Switch branches and do your work
git switch other-branch
Step 3: Return to original branch
git switch original-branch
Step 4: Restore your work
git stash pop
View stashes: git stash list

Sync Fork with Original Repo

Medium
Scenario: Your fork is behind the original repository.
Step 1: Add upstream remote (once)
git remote add upstream <original-repo-url>
Step 2: Fetch upstream changes
git fetch upstream
Step 3: Merge upstream into your branch
git merge upstream/main
Step 4: Push updates to your fork
git push origin main

Remove Untracked Files

Easy
Scenario: You have generated files or build artifacts to clean up.
Step 1: Preview what will be deleted
git clean -n
Step 2: Delete untracked files
git clean -f
Step 3: Also remove directories
git clean -fd
Warning: git clean permanently deletes files. Always preview with -n first!

Clean Up Commit History

Hard
Scenario: You have multiple messy commits that should be squashed before merging.
Step 1: Start interactive rebase
git rebase -i HEAD~3
Replace 3 with number of commits to edit
Step 2: In the editor, change 'pick' to 'squash' 
pick abc123 First commit
squash def456 Fix typo
squash ghi789 Another fix
Step 3: Save and edit the combined commit message
Step 4: Force push if already pushed
git push --force-with-lease
Warning: Never rebase commits that others are working on!

Recover Lost Commits

Hard
Scenario: You did a hard reset or rebase and lost commits you need.
Step 1: View reflog (history of HEAD)
git reflog
Step 2: Find the commit you want
Look for the commit hash before the mistake
Step 3: Create a branch at that commit
git branch recovery <commit-hash>
Step 4: Or reset to that commit
git reset --hard <commit-hash>
Reflog saves you: Git keeps reflogs for 90 days by default, so lost commits can usually be recovered!

Fix Wrong Author on Commit

Medium
Scenario: You committed with the wrong name/email.
For the last commit only:
git commit --amend --author="Name <email@example.com>"
Set correct user for this repo:
git config user.name "Your Name"git config user.email "email@example.com"
If already pushed:
git push --force-with-lease

Move Specific Commits to Another Branch

Medium
Scenario: You want specific commits from one branch on another branch.
Step 1: Find commit hashes
git log --oneline
Step 2: Switch to target branch
git switch target-branch
Step 3: Cherry-pick the commits
git cherry-pick <commit-hash>
For multiple commits:
git cherry-pick <hash1> <hash2> <hash3>
